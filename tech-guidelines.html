Standards Techniques

Ce document détaille les principes d'ingénierie front-end utilisés pour KeurGui. L'objectif est un code léger, performant sur les réseaux 3G/4G, et maintenable sans framework lourd.

1. Philosophie "Vanilla First"

Nous n'utilisons pas de Bootstrap, Tailwind ou Bulma par défaut. Pourquoi ?

Performance

Le CSS généré est ultra-léger (< 50kb). Essentiel pour le chargement mobile au Sénégal.

Maintenabilité

Pas de dépendance externe cachée. Un développeur junior avec des bases HTML/CSS peut tout comprendre.

2. Stratégie Mobile First

C'est la règle d'or. Nous écrivons le CSS pour le mobile par défaut, puis nous ajoutons des règles pour les écrans plus larges.

L'approche technique (Media Queries)

On utilise min-width et jamais max-width pour la structure.

/* ❌ MAUVAISE PRATIQUE (Desktop First) */
.card { width: 50%; } /* On pense PC d'abord */
@media (max-width: 768px) { .card { width: 100%; } } /* Puis on surcharge pour mobile */

/* ✅ BONNE PRATIQUE (Mobile First - KeurGui Standard) */
.card { width: 100%; } /* Par défaut, c'est pour mobile (léger) */
@media (min-width: 768px) { .card { width: 50%; } } /* On adapte si l'écran est grand */

3. Variables CSS (Custom Properties)

Nous utilisons :root pour définir les tokens de design. Cela permet de changer le thème (ex: Dark Mode) instantanément sans toucher au HTML.

:root {
  --primary: #0E3B43;
  --radius: 16px;
  --spacing-unit: 1rem;
}

.btn {
  background-color: var(--primary);
  border-radius: var(--radius);
}

4. Layout : Grid vs Flexbox

Nous ne flippons pas de pièces. Chaque outil a son usage strict dans le projet.

CSS Grid

Utilisé pour la structure globale de la page (Sidebar + Main) et les grilles 2D (Galerie photos, Liste de cartes).

display: grid;
Flexbox

Utilisé pour l'alignement 1D à l'intérieur des composants (Boutons avec icônes, Navbar, Badges).

display: flex;

5. Typographie Fluide (Clamp)

Plutôt que d'écrire 5 media queries pour changer la taille de police, nous utilisons la fonction mathématique clamp().

/* La police fera minimum 2rem, idéalement 5% de l'écran, maximum 4rem */
h1 {
  font-size: clamp(2rem, 5vw, 4rem);
}

Cela garantit que les titres sont toujours lisibles, que ce soit sur un iPhone SE ou un écran 4K.

6. Bonnes Pratiques Code

À Faire (Do)
  • Utiliser HTML5 sémantique (<nav>, <header>, <main>).
  • Utiliser rem pour les tailles de police et marges (accessibilité).
  • Nommer les classes en Anglais (standard mondial).
  • Commenter les hacks CSS complexes.
À Ne Pas Faire (Don't)
  • Ne jamais utiliser !important (sauf cas de force majeure).
  • Ne pas utiliser de styles inline (style="...") dans le HTML.
  • Ne pas utiliser height fixe sur des conteneurs de texte (ça casse si le texte est traduit).
  • Ne pas utiliser px pour les polices.
